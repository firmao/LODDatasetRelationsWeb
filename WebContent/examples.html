Examples

An example of how useful is this index can be the following: Given the following SPARQL query that works on CKAN SPARQL endpoint https://linked.opendata.cz/sparql:

Select * where { ?s ?p ?o.
    Filter(?s=<http://purl.org/dc/terms/date> ||
    ?s=<http://crime.rkbexplorer.com/id/location> ||
    ?s=<http://purl.org/dc/terms/subject>
)}

Thanks to the index we are able to know that we can also execute the query at https://eu.dbpedia.org/sparql but not at http://dbpedia.org/sparql, and we can execute at least in more 5 datasets also complementing the results.

Filtering datasets by class:

Note: Here we consider class also as a concept or type of entity.

For instance, if you want all the datasets containing a given class.

Example: All datasets containing information about the class city that are similar to DBpedia dataset.

You just need to provide the URI of the required class/concept/type, in this case, the input will be:

http://dbpedia.org/ontology/City

In this way is possible to know all datasets about city from LOD cloud.
